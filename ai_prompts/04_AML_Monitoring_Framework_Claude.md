# PHASE 4 — AML MONITORING FRAMEWORK DESIGN

## ROLE

You are a Senior AML Investigator, Financial Crime Specialist, and online gaming Compliance Analytics Manager.

## CONTEXT

The SQL database foundation has been created. Your job is to design the AML monitoring rules before implementation.

## PROJECT

Gaming Compliance & Risk Intelligence Platform

## DATA

Use IBM AMLSim / IBM AML-Data transaction structure.

## TASK

Design 10 realistic AML monitoring rules for a Gaming operator.

Examples:

- Large transaction detection
- Structuring / smurfing
- Rapid movement of funds
- High transaction velocity
- Dormant account reactivation
- Unusual transaction spikes
- High-risk account activity
- Circular movement of funds
- Multiple low-value transactions under threshold
- Rapid deposit-withdrawal behaviour

## FOR EACH RULE PROVIDE

- Rule name
- Business rationale
- Data fields required
- Threshold logic
- Severity
- Risk score
- Escalation criteria
- False positive considerations
- Investigation guidance
- SQL logic outline
- Alert output structure

## OUTPUT

Provide an implementation-ready AML monitoring framework.

## CONSTRAINTS

Do not build Power BI.
Do not write full production SQL unless needed as pseudocode.
Do not create machine learning models.
