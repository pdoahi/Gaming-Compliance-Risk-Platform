# data_raw

Raw source data files. Never overwrite or modify files in this folder.

## Expected Sources

- IBM AML-Data transaction CSVs

## Current Data (Synthetic)

This project currently ships **synthetic** data generated by
[`synthetic_data_generator.py`](synthetic_data_generator.py), which mirrors the IBM
*Transactions for AML* schema so the real dataset can be dropped in unchanged:

- `transactions_synthetic.csv` — ~5.4k transactions, ~7% labelled laundering,
  with injected structuring / large-transaction / rapid-movement / velocity /
  round-number / counterparty-concentration typologies.

To regenerate: `python data_raw/synthetic_data_generator.py`

## Market / GGR Data (Synthetic)

Also generated by [`synthetic_data_generator.py`](synthetic_data_generator.py)
(`generate_market()`), a fabricated monthly online-gaming market series for the
GGR-reporting layer. The figures are **illustrative only** and do not correspond to
any real market, operator, or regulator:

- `market_monthly_synthetic.csv` — 48 synthetic months: cash wagers, NAGGR (GGR),
  active player accounts, ARPPA.
- `market_by_product_synthetic.csv` — the same months split by product category
  (Casino / Sports / Poker).

To regenerate: `python data_raw/synthetic_data_generator.py`. The schema is kept
realistic so a real, properly-licensed market dataset could be substituted later.

## Swapping in Real Data

Download the IBM/Kaggle AML CSV, rename its columns to match
`transactions_synthetic.csv`, and place it here — the notebooks and SQL staging
will work without modification.

## Important

Synthetic files are small and committed for reproducibility. Do not commit large
real datasets — add them to `.gitignore` and document their source here instead.
